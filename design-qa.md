# Design QA

Final result: passed

## 验证范围

- GitHub Pages 静态站入口：`index.html`
- V1 左侧菜单：项目管理、采购管理、合同管理、项目进度、付款申请、发票管理、验收管理、项目结项、系统管理
- V1 主流程：项目申请 -> 审批 -> 采购 -> 合同 -> 项目实施 -> 付款申请 -> 发票登记 -> 验收 -> 结项
- 核心交互：筛选、重置、新增、编辑、详情、审批抽屉、二次确认、PRD 弹窗、流程图弹窗、列表展示

## 浏览器检查结果

- 已按 V1 功能重新生成前端页面并覆盖旧内容。
- 所有业务页面顶部的 V1 概览、关键指标与流程条区域已删除，页面直接进入标签、筛选与列表。
- 项目管理页面已调整为两个标签：项目管理、项目审批；项目管理支持创建项目和查看详情，项目审批支持审批流程处理。
- 项目管理标签页右上角已新增“项目类型管理”按钮，点击进入项目类型管理子页面；子页面支持新建、编辑、删除项目类型，已使用项目类型点击删除时提示“已使用项目类型不支持删除操作”。
- 项目类型管理子页面已精简展示：顶部仅保留“返回”，列表不展示编号、类型编码、使用项目数、状态和左侧选择框，仅展示类型名称、创建时间、创建人员和操作。
- 项目类型管理新增/编辑弹窗仅保留类型名称与类型说明；创建时间按应用首次打开时间生成，创建人员固定为 admin。
- 未使用项目类型删除弹窗已隐藏“二次确认”小字和中间详情框，仅保留确认文案与取消/确认操作。
- 所有二次确认弹窗已统一隐藏顶部“二次确认”字段和中间摘要信息块，仅保留标题、说明文案、取消与确认操作。
- 项目管理列表点击详情后进入项目详情子页面，不再打开右侧抽屉；详情页“V1项目时间轴”已改为“进度表”，展示节点名称与完成时间，未完成节点显示 `-`。
- 项目周期已改为日期范围选择，精确到日；项目状态集合已删除“草稿”，新建项目默认进入“待审批”。
- 项目审批列表已删除当前节点、审批角色、审批顺序、审批记录字段；点击审批后，审批面板上半部分展示与项目详情一致的项目内容，下方展示审批流程和审批操作。
- 项目审批页面已删除“处理审批”和“导出”操作按钮；点击审批后进入审批详情子页面，不再打开右侧抽屉。
- 项目详情与审批详情的附件字段支持多附件展示，每个附件提供“预览”操作并显示预览反馈。
- 采购管理顶部采购状态说明条已删除；采购列表新增实际采购金额、招标公告发布时间、投标截止时间、中标单位确定时间字段。
- 点击“开始采购”会弹出表单要求填写实际采购金额，并可记录采购关键环节时间；确认后采购状态进入采购中，同时同步更新项目进度。
- 项目类型、类型、状态类筛选项已调整为下拉选择框；状态选项来自当前页面列表数据，普通文本字段继续使用输入框。
- 采购管理已合并为单一“采购登记”页面，不再保留重复的“采购跟踪”标签。
- 采购状态说明已补充：待采购、采购中、采购完成分别对应登记后未启动、执行中、文件与供应商确认完成。
- 采购状态动作按当前状态显示：待采购显示“开始采购”，采购中显示“完成采购”，采购完成不再显示推进按钮。
- 列表数据不超过 10 条时不显示分页控件，避免为分页而强行分页。
- 9 个左侧菜单均可点击并展示模拟业务数据。
- 新增/编辑弹窗默认不带业务数据，仅显示“请输入 / 请选择 / 点击上传”等提示。
- 必填字段仅在字段名前展示 `*`，字段后无“必填 / 非必填”文字。
- 审批列表仅保留“审批”操作，审批抽屉展示申请列表信息、审批流程和同意/驳回/退回修改操作。
- PRD 绿色点可打开当前页面 PRD，内容包含 PRD-V1 九段结构。
- 左下角“流程图”可打开 V1 全流程、项目管理、采购合同、付款发票、验收结项、系统配置流程。
- 关键动作均先弹出二次确认，再更新模拟状态。
- `index.html` 中文标题、按钮、弹窗文案显示正常，无乱码。
- 页面运行未捕获 JavaScript 错误。
