# Design QA

final result: passed

## 验证范围

- GitHub Pages 静态站入口：`index.html`
- 企业级分组折叠菜单：11 个一级业务分组，51 个左侧业务页面
- 页面覆盖：工作台、项目、采购、合同、实施、付款、发票、验收、结项、统计分析、系统管理
- 关键交互：左侧页面切换、新增弹窗、PRD 弹窗、流程图弹窗、审批二次确认、状态更新

- 左侧导航可访问页面数：51

- 页面示例：
  - 项目申请、项目审批、项目详情、项目变更、项目归档
  - 采购申请、采购审批、招标管理、供应商管理、采购进度
  - 用户管理、角色权限、流程管理、表单配置、系统设置
- 新增弹窗必填星号正常显示。
- 新增弹窗未出现字段后的“必填”文字。
- PRD 弹窗包含 PRD-V1 结构。
- 流程图弹窗包含 6 组业务/操作流程。
- 审批通过操作会先弹出二次确认，确认后状态更新为“已通过”。
- 页面运行未捕获 JavaScript 错误。

## 视觉核验

- 页面延续参考图的浅色后台风格、绿色主色、小尺寸按钮、紧凑筛选区和低对比表格。
- 工作台提供数据概览、待办、我的项目、消息通知和预算执行驾驶舱。
- 统计分析的可视化大屏使用模拟图表表达，不引入第三方库，适配静态部署。
